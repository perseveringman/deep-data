## Summary

Agent 技术的发展演进历经提示词工程、上下文工程与 Harness 工程三个核心阶段，标志着智能体从单纯的指令理解向系统化、稳定化运行的范式转变。提示词工程通过约束语义空间激发模型已有能力；上下文工程利用 RAG 和建议式披露技术扩展模型知识边界；而 Harness 工程则引入工具系统、任务编排、记忆管理、评估观测及故障恢复机制，将大模型这一随机系统转化为可控的稳定系统。以 Claude 为代表的现代智能体架构，通过分级提示词管理、统一工具安全规范及多层记忆压缩策略，实现了从 “聪明” 到 “靠谱” 的进化，确保在复杂任务执行中具备高可靠性与工程化落地能力。

## Takeaways

1. Agent 的发展历程是从 “提示词工程”（指令优化）到 “上下文工程”（信息输入工程化），最终演进为 “Harness 工程”（系统运行工程化）的系统性跨越。
2. 提示词本质上并非赋予模型新知识，而是通过约束语义空间来激发模型已有的潜在能力，起到的是 “限制” 而非 “创造” 的作用。
3. 智能体开发的核心逻辑已从追求 “智商”（模型聪明程度）转向追求 “稳定”（系统可靠性），这与个体从儿童成长为成年人时对 “靠谱” 的评价标准转变高度一致。
4. Harness 架构通过引入读写状态规范、破坏性确认及并发控制这三大安全维度，将大模型这一 “随机系统” 转化为可控的 “稳定系统”。
5. 上下文工程中的 “建议式披露” 技术（如 Skill 机制），通过按需加载工具摘要而非全量加载说明书，有效解决了长文本导致的注意力涣散和窗口稀缺问题。
6. Claude 的记忆系统通过 “白名单” 机制，将记忆划分为用户偏好、状态反馈、项目记录及外部指针四类，实现了对关键信息的精准存储与冗余信息的有效剔除。
7. 智能体编排中的 “三重奏” 模式（规划器、生成器、评估器）本质上是对传统软件开发流程的 AI 化重构，通过反向迭代确保了代码输出的高质量。
8. 针对智能体运行中的 “翻车” 风险，建立自动重试、状态回滚及人工介入的兜底机制，是实现智能体从 Demo 走向生产环境的必要工程前提。
9. 提示词分级管理（通用红线与个性化动态配置）不仅提升了管理效率，还通过缓存机制显著降低了模型调用的成本与延迟。
